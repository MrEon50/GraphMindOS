# 🧠 GraphMindOS v1.0 [Evolutionary Release]
**Autonomous, Human-UI-Free, AI-Native Operating System**

GraphMindOS to eksperymentalny ekosystem, w którym AI przestaje być tylko "chatbotem", a staje się autonomicznym operatorem systemu. To OS zaprojektowany dla agentów, gdzie logika nie jest wpisana na sztywno w kodzie, lecz kompilowana w locie do formy **Dynamicznych Grafów Akcji (DAG)**. System natywnie wykorzystuje lokalne modele AI, zapewniając 100% prywatności i kontroli.

## 🚀 Kluczowe Funkcjonalności v1.0

1.  **Autonomous Tool Factory (Self-Evolution):** AI potrafi projektować i wstrzykiwać do swojego rdzenia nowe "Atomy" (narzędzia) w Pythonie, odpowiadając na potrzeby użytkownika.
2.  **Hybrid RAG Memory:** Pełna integracja z wektorową bazą danych. Każda interakcja, odczytany plik czy strona WWW trafia do pamięci długoterminowej, budując bazę wiedzy systemu.
3.  **Network Senses:** System posiada narządy do wyszukiwania informacji w sieci oraz głębokiego czytania treści stron i wyodrębniania czystego tekstu.
4.  **Human-in-the-Loop Safety:** Zaawansowany moduł autoryzacji z funkcją analizy konsekwencji. AI ostrzega przed ryzykiem operacyjnym, a ostateczna decyzja [Y/N] zawsze należy do Architekta.
5.  **Dynamic Tool Awareness:** Kompilator Intencji w czasie rzeczywistym skanuje dostępne narzędzia i uczy się z nich korzystać, adaptując się do nowych możliwości bez konieczności restartu.

## 📂 Anatomia Systemu

-   **Fabryka Narzędzi** – Moduł odpowiedzialny za autonomiczne generowanie kodu i rozszerzanie możliwości systemu w locie.
-   **Magazyn Semantyczny** – Warstwa zarządzająca pamięcią długoterminową (RAG) oraz procesami wektoryzacji wiedzy.
-   **Kompilator Intencji** – Główny procesor tłumaczący naturalny język człowieka na topologie sterujące i grafy akcji.
-   **Silnik Topologii** – Serce egzekucyjne nawigujące po strukturze grafu i uruchamiające wybrane narządy wykonawcze.
-   **Repozytorium Ewolucyjne** – Bezpieczna przestrzeń zapisu i ładowania nowych, autorskich wtyczek stworzonych przez AI.

## 🛠️ Instalacja i Uruchomienie

### Wymagania
* Środowisko lokalnych modeli LLM (obsługujące API na porcie 11434).
* Python 3.9+
* Biblioteki: `pydantic`, `chromadb`, `beautifulsoup4`, `requests`.

### Start Systemu
1. Zainstaluj wymagane pakiety: `pip install pydantic chromadb beautifulsoup4 requests`
2. Uruchom główny moduł: `python main.py`
3. Wybierz model z listy i zacznij wydawać Intencje.

## 💡 Przykłady Użycia

*   **Ewolucja:** `Stwórz atom, który sprawdzi aktualne zużycie zasobów systemowych.`
*   **Research:** `Znajdź w sieci informacje o nowościach w świecie AI i zapisz raport w workspace/news.txt.`
*   **Zarządzanie:** `Odczytaj plik konfiguracyjny i na jego podstawie stwórz podsumowanie w pamięci RAG.`
